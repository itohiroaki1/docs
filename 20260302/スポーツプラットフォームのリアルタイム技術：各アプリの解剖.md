# スポーツプラットフォームのリアルタイム技術：各アプリの解剖

**作成日:** 2026年3月2日

---

## 概要：「リアルタイム性」とは何か

スポーツ観戦においてリアルタイム性は、単なる「速さ」ではなく、**視聴体験の根幹**をなす要素です。試合の結果をSNSで知ってしまう「ネタバレ問題」、ベッティングにおけるオッズの公平性、ファンタジースポーツのスコア更新——これらすべてに「どれだけ遅延なく情報を届けられるか」が直結します。

遅延の種類は大きく二つに分けられます。**映像の遅延**（放送遅延）と**データの遅延**（スコア・統計情報の更新遅延）です。前者は数秒〜数十秒のオーダー、後者はミリ秒〜数秒のオーダーで争われます。以下では、前述のレポートに登場した各アプリケーションが、それぞれどの種類のリアルタイム性を追求し、どのような技術で解決しているかを詳述します。

## 2. 各アプリの技術的解決策

### 2.1. Dream11（インド）：5000万人同時接続を支えるリアルタイムランキングエンジン

Dream11が直面する技術的課題は、スポーツアプリの中でも最も過酷な部類に入ります。IPLファイナルでは、**1球ごとに4000万件のレコードを60秒以内に処理し、5000万人以上の同時接続ユーザーに一貫したランキングを提供する**という要件があります。これは毎秒66万7000件のレコードを処理することに相当します。

#### 採用技術と解決策

Dream11が採用したのは、**スナップショット分離アーキテクチャ（Snapshot Isolation Architecture）**と呼ばれる独自の設計思想です。

**データパイプラインの全体像：**

```
ユーザー操作 → MySQL（トランザクション処理）
    ↓
Amazon S3（データ抽出）→ SNS通知 → Lambda（オーケストレーション）
    ↓
Apache Spark クラスター（並列バッチ処理）
    ↓
Apache Cassandra（スケーラブルな読み書き）
    ↓
各Webサーバーのローカル Redis（分散キャッシュ）
    ↓
ユーザー（<1msのレスポンス）
```

**スナップショット分離の仕組み：** 従来のシステムがデータを「上書き更新」するのに対し、Dream11は1球ごとに**不変（イミュータブル）なスナップショット**を生成します。全てのビュー（リーダーボード、チームビュー、個人スタッツ）が同一のスナップショットバージョンを参照するため、ユーザーが異なる画面を見ても矛盾が生じません。これはデータベースの**MVCC（多版型同時実行制御）**の概念をアプリケーション層全体に拡張したものです。

**Apache Sparkによる「全か無か」の保証：** 20ノードのSparkクラスター（合計40GBの分散メモリ）が4000万件のレコードを並列処理します。Sparkはジョブ全体を単一の処理単位として扱い、途中でノードが故障しても自動的にタスクを再スケジュールします。処理が完全に成功しない限り、一切のデータ更新を行わない「All or Nothing」の保証を実現しています。

**Apache Cassandraによる高可用性：** 書き込みが集中するスコア更新には、マスター・スレーブ構成ではなく、**リング型の分散アーキテクチャ**を持つCassandraを採用。どのノードも読み書きを処理でき、複数ノードへの同時書き込みによってレプリケーション遅延を排除しています。

**分散キャッシュによる超低遅延：** 中央集権型のRedisクラスターを廃し、各Webサーバーに**ローカルのRedisキャッシュ**を配置。スナップショットが不変であるため、複雑なキャッシュ無効化ロジックが不要になり、**キャッシュヒット時の応答時間は1ミリ秒未満**を実現しています。

### 2.2. JioCinema（インド）：世界記録3200万同時視聴者を支えた配信インフラ

2023年のIPLファイナルで、JioCinemaは**3200万人の同時視聴者**という世界記録を樹立しました。この規模は、単一のクラウドプロバイダーのリソースだけでは対応できないレベルです。

#### 採用技術と解決策

**HLS + CDNの組み合わせ：** 映像はHLS（HTTP Live Streaming）形式で、4〜6秒のセグメントに分割されます。これらのセグメントを世界中のCDN（コンテンツデリバリーネットワーク）のエッジサーバーにキャッシュすることで、ユーザーの近くから映像を配信し、遅延とバックエンドへの負荷を最小化します。

**アダプティブビットレートストリーミング（ABR）：** ユーザーの回線速度に応じて、自動的に映像品質（解像度）を切り替えます。インドでは農村部と都市部で通信インフラの格差が大きいため、このABR技術が特に重要です。

**カスタムオートスケーラー：** 一般的なクラウドのオートスケーリングは、急激なトラフィック増加に対応するには遅すぎます。JioCinemaは、同時視聴者数（コンカレンシー）という単一の指標に基づいて全サービスを一括でスケールする**カスタムスケーリングシステム**を構築しました。試合開始前から事前にリソースを確保し、急激なアクセス増加に備えます。

**クラウドリソースの限界と対策：** 大規模イベント時には、特定のリージョンのクラウドリソースが物理的に枯渇する可能性があります。そのため、前年末から複数のインフラプロバイダーと協議し、年間を通じた**容量計画（キャパシティプランニング）**を実施します。

### 2.3. Amazon Prime Video「Thursday Night Football」（アメリカ）：AIが試合をリアルタイムで解析する

Amazon Prime Videoは、NFLの「Thursday Night Football（TNF）」の独占配信において、単なる映像配信を超えた**AIによるリアルタイム試合解析**を視聴体験に組み込んでいます。

#### 採用技術と解決策

**NFL Next Gen Stats（NGS）のデータ収集：** 全22選手のショルダーパッドとフットボール自体に**RFIDチップ**が埋め込まれており、スタジアムに設置された20以上のウルトラワイドバンド受信機が、選手の位置を**毎秒10回（ボールは毎秒25回）**、数センチの精度で計測します。

**AWSによるリアルタイムML処理：** 収集されたトラッキングデータは、AWS SageMakerで動作する**75以上の機械学習モデル**によって1秒以内に処理されます。これにより、1プレーあたり500〜1000種類の統計情報がリアルタイムで生成されます。

**Prime Visionの主なAI機能（2025年シーズン）：**

| 機能名 | 内容 |
| :--- | :--- |
| **Pocket Health** | AIがOラインの数万のデータポイントを瞬時に分析し、QBのポケットへの脅威レベルを視覚化 |
| **Defensive Alerts** | スナップ前の守備選手の動きを解析し、ブリッツの可能性が高い選手をリアルタイムでハイライト |
| **Coverage ID** | スナップ前にディフェンスのスキーム（ゾーン/マン）をリアルタイムで特定 |
| **Path to Victory** | 試合終盤、劣勢チームの逆転シナリオをAIが予測・可視化 |
| **Prime Targets** | レシーバーのルート上でオープンになった瞬間をリアルタイムで緑のオーブでハイライト |

これらの機能は、放送中の映像に**オーバーレイ表示**されるため、視聴者はテレビ解説者と同等の分析情報を、試合の流れを止めることなくリアルタイムで享受できます。

### 2.4. DraftKings / FanDuel（アメリカ）：ライブベッティングを支えるサブ秒データフィード

スポーツベッティングにおけるリアルタイム性は、**金銭的な公平性**に直結するため、最も厳格な要件が求められます。1秒の遅延が、情報を先に入手した者による不正な賭けを可能にしてしまうからです。

#### 採用技術と解決策

**Sportradarのリアルタイムデータフィード：** DraftKingsやFanDuelのようなベッティングプラットフォームは、**Sportradar**などのスポーツデータプロバイダーから、**サブ秒（1秒未満）のレイテンシ**でリアルタイムデータを受信します。Sportradarは、AIを活用したデータ収集システムにより、試合の進行状況をほぼリアルタイムで処理・配信します。

**WebSocketによるプッシュ型通信：** オッズの更新は、ユーザーがリクエストを送るたびにサーバーが応答する「ポーリング」方式ではなく、サーバーからクライアントへデータを能動的に送信する**WebSocket**によるプッシュ型通信で実現されています。これにより、オッズが変動した瞬間にユーザーの画面が自動更新されます。

**Hawk-Eye / SkeleTRACKとの連携：** ソニー傘下のHawk-Eyeが開発した**SkeleTRACK**は、スタジアムに設置された複数のカメラが捉えた映像を、AIと機械学習モデルでリアルタイムに処理し、選手29箇所の骨格データとボールの3D座標を生成します。このデータはApache KafkaとApache Flinkを組み合わせたストリーミングパイプラインで処理され、ベッティングプラットフォームを含む下流のシステムへとサブ秒で配信されます。

### 2.5. DAZN（イギリス）：年間11万件以上のイベントを支える低遅延配信基盤

DAZNは200以上の市場で年間11万件以上のライブイベントを配信する、世界最大規模のスポーツストリーミングサービスです。

#### 採用技術と解決策

**Bitmovinとの協業によるLL-HLS / LL-DASH：** DAZNは動画エンコーダー・プレーヤーのBitmovinと協業し、**Low-Latency HLS（LL-HLS）**および**Low-Latency DASH（LL-DASH）**を採用しています。従来のHLSが15〜30秒の遅延を生じさせるのに対し、LL-HLSはセグメントをさらに細かく分割（チャンキング）することで、**3〜5秒程度の遅延**を実現します。

**プライベートエッジCDNアーキテクチャ：** DAZNは大手CDNプロバイダーに依存するだけでなく、ISP（インターネットサービスプロバイダー）のコアネットワーク内に**プライベートエッジCDN**を構築しています。これにより、遅延に敏感なライブ映像が、一般のインターネットトラフィックと同じ混雑した経路を通らずに配信されます。

**次世代プロトコル「MoQ」への移行：** 現在、業界全体で注目されているのが、IETFが標準化を進める**MoQ（Media over QUIC）**です。Cloudflareが2025年8月に世界330都市以上のデータセンターでMoQリレーネットワークの運用を開始しました。MoQは、QUICプロトコルを基盤とすることで以下の課題を解決します。

| 課題 | 従来技術（TCP/HLS） | MoQ（QUIC基盤） |
| :--- | :--- | :--- |
| **遅延** | 15〜30秒（標準HLS） | **サブ秒（1秒未満）** |
| **パケットロス時の挙動** | ヘッドオブラインブロッキング（全体が止まる） | **独立したストリームで影響を局所化** |
| **ネットワーク切替** | 再接続・再バッファリングが必要 | **シームレスな接続マイグレーション** |
| **スケーラビリティ** | WebRTCはN²問題でスケール困難 | **CDNスケールでサブ秒遅延** |

MoQは映像だけでなく、スポーツのスコア更新のような低帯域幅データの配信にも応用でき、スポーツプラットフォームの次世代インフラとして期待されています。

### 2.6. 中国「超視点（Ultra-Perspective）」システム：リアルタイム3D再構成

2025年11月の第15回全国運動会（中国）で初披露された「超視点」システムは、スポーツ映像のリアルタイム性を全く新しい次元に引き上げました。

#### 採用技術と解決策

**マルチカメラアレイ + リアルタイム3D再構成：** 会場の複数箇所に設置されたカメラアレイが、バスケットボールやバドミントンの試合を多角度から同時撮影します。この映像データを**ディープラーニングモデル**がリアルタイムで処理し、競技フィールド全体の**完全な3D仮想モデル**を生成します。

**自由視点映像（Free Viewpoint Video）：** 生成された3Dモデルを基に、視聴者はスマートフォンやタブレット上で**360度任意の角度にカメラを動かして**試合を観戦できます。特定の選手を追いかけたり、ボールの軌跡を真上から確認したりすることが可能です。

**技術的課題と解決：** この技術の最大の難しさは、複数カメラの映像を同期させ、3Dモデルを生成し、それをユーザーのデバイスに届けるまでの全工程を、視聴者が「リアルタイム」と感じられる遅延内（概ね数秒以内）で完結させることです。中央電視台（CCTV）、国家重点実験室、上海・広東の専門チームが連携して開発した独自システムがこれを実現しています。

       （数千人）                    （数千万人）

## 4. 今後の技術的展望

### 4.1. MoQ（Media over QUIC）の普及

2025年にCloudflareが本格展開を開始したMoQは、映像配信の「次世代標準」として業界全体に普及していく見込みです。WebRTCの低遅延とHLS/DASHのスケーラビリティを両立するこのプロトコルが広まれば、スポーツ中継における「ネタバレ問題」は技術的に解決されます。

### 4.2. エッジコンピューティングとAIの融合

5Gの普及に伴い、スタジアムや放送局の近くに設置された**エッジサーバー**でAI処理を行うことで、クラウドへのデータ往復時間を削減し、さらなる低遅延化が実現します。Hawk-EyeのSkeleTRACKは既にクラウドとオンプレミスの両方に対応しており、この方向性を先取りしています。

### 4.3. 「超視点」技術のグローバル展開

中国で実証された自由視点映像技術は、5Gの高帯域幅と低遅延を活かすことで、スタジアムの外にいるファンにも「スタジアムの中を自由に歩き回るような」体験を提供できます。この技術がグローバルに普及すれば、スポーツ観戦体験は根本的に変わるでしょう。
